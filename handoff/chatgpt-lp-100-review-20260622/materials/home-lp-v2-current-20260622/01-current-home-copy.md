## BEFORE / AFTER

H2:

```text
コピペ、話者分け、改行直し。一本ごとの繰り返しを減らす。
```

Body:

```text
コメントを拾う。不要な部分を外す。話者を割り当てる。改行と文字量を整える。素材の場所を確認する。最後にYMM4でもう一度組み直す。ゆっくりまとめプロセッサーは、この繰り返しを一つの制作データとしてつなぎます。
```

比較:

```text
手作業: ページを行き来して本文やコメントをコピー
本製品: 対応URLから取得候補を確認して取り込む

手作業: 話者、タイトル読み、改行を毎回設定
本製品: よく使うルールをプリセットで呼び出す

手作業: 台本と素材フォルダを何度も往復
本製品: ボードで素材の位置と不足を確認する

手作業: YMM4で同じ内容を一から並べ直す
本製品: 確認した台本と素材をYMM4へ反映する
```

## Workflow

H2:

```text
ネタを入れる。台本を整える。YMM4へ反映する。
```

Body:

```text
どこをアプリに任せ、どこを自分で確認するかが分かる流れです。AIに丸投げせず、最後の品質は自分で決められます。
```

Step 01:

```text
URLでも、下書きでも始められる。
対応する記事・スレッドのURL、手元の下書き、テーマから開始。URL取得では候補を確認し、使う本文やコメントを選びます。
```

Step 02:

```text
話者・改行・タイトルを、動画の型に合わせる。
話者の順番、タイトル読み、改行、禁止語変換、コメント取得などを設定。よく使う組み合わせはプリセットとして次の一本でも呼び出せます。
```

Step 03:

```text
AIの出力条件を決め、結果を見てから使う。
構成、口調、折り返し、テーマ、目標文字数を指定して台本案を生成。プレビューで内容を確認し、必要な箇所を直してから台本編集へ送ります。
```

Step 04:

```text
素材を確認し、YMM4へ反映する。
台本と素材の配置を確認してYMM4へ反映。音声、字幕、間、演出、出典、公開前の最終確認はYMM4で行います。
```

CTA:

```text
Free版でこの流れを試す
```

## Product Features

Feature 01:

```text
必要な本文とコメントだけ、取り込む。
サイトとURLを指定すると、取得候補を一覧で確認できます。本文やコメントをそのまま流し込まず、使う内容を見てから取り込めます。
```

Feature 02:

```text
いつもの話者・改行・構成を、プリセットで呼び出す。
タイトル読み、話者の順番、改行、禁止語変換、構成、文字量。チャンネルでよく使う設定をプリセットにして、次の一本でも呼び出せます。
```

Feature 03:

```text
素材の抜けを確認して、YMM4へ反映する。
台本と素材の位置をボードで確認し、保存先や素材パスの行き来を減らします。確認した内容をYMM4へ反映し、最後はタイミングと演出に集中できます。
```
