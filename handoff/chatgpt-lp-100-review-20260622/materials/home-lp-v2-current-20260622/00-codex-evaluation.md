# Codex評価

## 総合評価

現時点では **78 / 100** くらいです。

「売るページ」としての骨格はかなり良くなりました。
v1の「購入前の注意を整理したページ」から、v2では「記事・スレッドからYMM4のタイムラインまで」という商品価値を前に出すページへ変わっています。

## 良いところ

1. Heroで商品理解が速い
   - 入力: 記事・スレッド
   - 処理: 本文/コメント取得、話者割り当て、台本整形、素材確認
   - 到達: YMM4反映
   - 条件: Windows 10 / 11、YMM4必須、Premium 39,800円、買い切り

2. 誤認防止と販売コピーのバランスが良くなった
   - 「できないこと」を前面に出しすぎず、料金・仕様・FAQへ整理できています。
   - それでも、動画完成、投稿、権利確認、成果保証の範囲は明示しています。

3. Freeの役割が明確になった
   - ただの起動確認ではなく、台本編集、素材確認、YMM4連携の流れを確かめる入口として説明できています。

4. Premiumの理由が少し見えるようになった
   - 「制限解除」だけでなく、台本取得、AI台本生成、動画作成の継続利用という文脈に置けています。

5. UIはv2の方向に寄った
   - ゴールド発光や抽象背景を避け、青CTAと実画面中心へ寄せました。
   - PC/モバイルとも横スクロールや大きな崩れはありません。

## 弱いところ

1. 実画面素材の証拠力がまだ暫定
   - 現行は既存スクリーンショットの安全なクロップです。
   - 同じ架空プロジェクトで入力からYMM4反映まで一貫した証拠にはまだなっていません。

2. YMM4反映画面の実証が弱い
   - リポジトリ内に本当の「YMM4へ反映されたタイムライン」素材がありません。
   - 現行は「反映前確認」「YMM4へ反映する流れ」として誤認を避けています。

3. Free/Premiumの具体上限がまだ弱い
   - 仕様書は具体値表示を要求しています。
   - ただし現時点で確定値がないため、LPでは「Free利用枠」「利用制限解除」に留めています。
   - ここは商品側の事実が確定すると一段強くなります。

4. デモ動画は本撮影ではない
   - 旧抽象デモは使っていません。
   - ただし現行の `home-demo-60s.mp4` はスクリーンショットベースの軽量デモです。
   - 本番では実操作収録に差し替えるのが理想です。

5. もっと欲しくなる感情の山は作れる
   - 現行は堅実で誠実です。
   - 「毎回のコピペから解放される」感情価値は、Final CTAやBefore/Afterでもう少し強められます。

## 公開判断

暫定公開なら可能です。
ただし本気でCVRを取りにいくなら、公開後すぐに次の改善を入れたいです。

優先度順:

1. 同一架空プロジェクトで実画面を撮り直す
2. YMM4反映後タイムラインのスクリーンショットを追加する
3. 60秒実操作デモを実収録へ差し替える
4. Free/Premiumの具体上限を確定して比較表へ入れる
5. HeroのA/Bテストを始める
