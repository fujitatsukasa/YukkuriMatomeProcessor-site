# 残課題と次アクション

## 重要度A

### 1. 同一架空プロジェクトで実画面を撮り直す

現行LPは既存スクリーンショットの安全なクロップを使っています。
公開品質を上げるには、同じ架空プロジェクトで次を一貫撮影してください。

```text
URL入力
取得候補
台本ルール
AI生成結果
台本編集
素材ボード
YMM4反映後タイムライン
Free状態
Launcher / 更新 / Premium状態
```

### 2. YMM4反映画面を本物へ差し替える

現行の `screen-home-ymm4-reflected-v1.webp` は、実YMM4の反映後タイムラインではありません。
誤認しないコピーへ調整済みですが、LPの証拠力としては弱いです。

必要な素材:

```text
screen-home-ymm4-reflected-v1.webp
```

条件:

- 実YMM4画面
- 台本・素材が反映済み
- 既存IP・実在ユーザー・個人情報なし
- YMM4で仕上げる前提が伝わる

### 3. 60秒デモを実操作収録へ差し替える

現行の `home-demo-60s.mp4` は、旧抽象デモを避けるためのスクリーンショットベース動画です。
本番では実アプリ操作と実YMM4だけで録画してください。

## 重要度B

### 4. Free / Premiumの具体上限を確定する

仕様書は具体値表示を要求しています。
現在は未確定値を作らず、公開可能な範囲で「Free利用枠」「利用制限解除」としています。

確定したい項目:

```text
URLからの台本取得
AI台本生成
台本編集
プリセット
素材ボード
プロジェクト保存
YMM4接続
YMM4反映
PC台数
アップデート
サポート
返金条件
```

### 5. 対応サイトの正式一覧を確定する

FAQでは「アプリ内のサイト選択とFAQで確認」と表現しています。
正式一覧が出せるなら、LP内のSpecsまたはFAQへ出すと購入前の不安が減ります。

### 6. 外部AI/APIのデータ送信説明を詳細化する

現行は機能によって通信先が変わる旨を記載しています。
プライバシーポリシー、FAQ、アプリ内表示と揃えて、送信内容を具体化すると信頼が上がります。

## 重要度C

### 7. Hero A/Bテスト

初期公開はv2確定コピーを使用中です。
アクセス数が取れたら次をテストできます。

```text
A: 記事・スレッドから、YMM4のタイムラインまで。
B: 次の一本は、台本のコピペから始めない。
```

CTAテスト:

```text
A: Free版をダウンロード
B: Freeで制作フローを試す
```

### 8. OG画像

`og-home-v1.webp` はまだ追加していません。
実アプリ画面を主役にして、細かい機能や価格を詰め込まないOG画像を作るとSNS共有時に強くなります。

- 既存IP・実在ユーザーコメントを証拠素材に使う
