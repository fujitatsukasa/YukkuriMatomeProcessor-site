# 04. 参考URLと真似る方向

目的は、既存LPの雰囲気を丸写しすることではなく、「画像の使い方」「プロダクト証拠の見せ方」「情報設計」を真似ること。

## 参考URL

### Descript

URL:

```text
https://www.descript.com/
```

見るところ:

- 動画編集/制作系ツールとして、抽象画像よりプロダクト体験を前に出す
- 何ができるかを短い言葉とビジュアルで見せる
- 生成っぽい謎画像ではなく、作業画面や出力イメージを使う

真似る方向:

- YMM4前提の実作業画面を主役にする
- 動画タイプ別に「どんな制作に使えるか」を見せる

### Screen Studio

URL:

```text
https://www.screen.studio/
```

見るところ:

- 画面録画/動画系ツールとして、プロダクト画面と完成物の見栄えを中心に訴求する
- 無駄な説明より、目で分かるデモが強い

真似る方向:

- 実機デモ動画をもっと目立たせる
- Home上で「触る前にどんな結果になるか」を分かるようにする

### Linear

URL:

```text
https://linear.app/
```

見るところ:

- ダークで上品
- 実UIスクショをきれいに見せる
- 余白、タイポグラフィ、モーションが控えめで精度高い

真似る方向:

- 画面カードの余白と見出しの品を上げる
- 実アプリスクショに注釈を載せる

### Notion

URL:

```text
https://www.notion.com/
```

見るところ:

- 抽象的な飾りより、具体的な使い道とテンプレート/画面を見せる
- ターゲット別の利用シーンが分かりやすい

真似る方向:

- 反応集、解説、ショート、掛け合いの用途別導線を強化する
- ユーザーの作業フロー別に見せる

### Raycast

URL:

```text
https://www.raycast.com/
```

見るところ:

- アプリ画面を高級感ある形で見せる
- 機能一覧より、仕事がどう速くなるかを示す

真似る方向:

- YMM4制作の面倒な作業を「どこが減るか」で見せる

### Cursor

URL:

```text
https://cursor.com/
```

見るところ:

- AI系でも、実プロダクト画面を中心にしている
- 抽象AI画像ではなく、操作・結果・コード/作業画面で見せる

真似る方向:

- AI台本生成は謎AIイメージではなく、生成結果の画面で見せる

## 画像方針まとめ

参考LPから採用すること:

- 実プロダクト画面
- 実デモ動画
- UI上の注釈
- 短い見出し
- CTA周辺の判断材料
- 料金/制限/前提を明確にする

採用しないこと:

- 3Dアイコンだけの説明
- 光る盾/矢印/謎パネル
- 架空UI
- 何を示しているか分からない抽象画像
- AI生成っぽいガラス素材
