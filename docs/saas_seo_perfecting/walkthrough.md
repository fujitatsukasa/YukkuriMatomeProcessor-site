## 2. 【最重要】SEO技術基盤の完全な刷新（CSRからSSGへ）
- **致命的弱点の克服:** これまでのサイトは完全にJavascriptで描画するReact SPAのため、インデックスされるまでにかかる「初速」が遅く、GoogleボットがJSを無効にしている場合に白紙のページ（`<div id="root"></div>`）しか返さない致命的な欠陥（SEOスコア上の85点）がありました。
- **Playwrightを用いた独自SSGの搭載:** `scripts/prerender.mjs` を開発。ビルドプロセス（`npm run build`）に割り込み、自前のローカルHTTPサーバーとPlaywrightヘッドレスブラウザを立ち上げて、sitemapにある15すべての経路をスクレイピングし、Reactが完全に描画し終えたあとの美しいDOMを静的HTMLファイル（SSG）として `dist/` 上に埋め込んで保存する仕組みへと変えました。
- **React Hydrationへの対応:**
  `main.tsx` に静的HTMLの検知機能を追加（`container.hasChildNodes()`判定）。これにより純粋なマウント（再描画のチラつき）ではなく、既存のDOMに対してイベントリスナーだけをスマートに繋ぎ合わせる「ハイドレーション（`hydrateRoot`）」を実装しました。
