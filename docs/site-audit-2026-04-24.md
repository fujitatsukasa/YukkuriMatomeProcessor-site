# サイト詳細評価 2026-04-24

対象: `vite-site/` の現行 Vite React サイト
確認環境: `http://127.0.0.1:5173/`
確認方法: production build、既存監査スクリプト、Playwright 実ブラウザ撮影、主要ページの desktop / mobile 目視確認

## 総合評価

総合判定: **B+ / 公開サイトとしての土台は強いが、トップページの知覚速度と購入導線の実務性に改善余地がある**

サイトの第一印象は、黒基調と金色アクセントで統一されており、個人開発ツールの紹介サイトとしてはかなり作り込まれている。単なる配布ページではなく、ダウンロード、使い方、FAQ、料金、ブログ、法務まで揃っているため、情報設計の骨格は十分に商用サイトの水準にある。

一方で、現在の強い演出と長大なページ構成は、初めて来たユーザーが「何ができるか」「自分に必要か」「どこから始めるか」を素早く判断するにはやや重い。特にモバイルのトップページはヒーロー表示までの導入演出が強く、初期状態では背景だけが長く見えるタイミングがある。見た目の完成度に比べて、即時理解と実務的な比較導線が少し遅れている。

- `/`
- `/download/`

## 良い点

### 情報設計

- 主要導線が `ホーム -> ダウンロード -> 使い方 -> FAQ -> 料金 -> 問い合わせ` と揃っている。
- ブログが単なる告知ではなく、YMM4、台本、素材、著作権、スマホ制作など検索流入を狙えるテーマになっている。
- 法務ページ、返金ポリシー、プライバシーポリシーまで用意されており、購入前不安への受け皿がある。
- ダウンロードページと使い方ページが分かれているため、既存ユーザーと初回ユーザーの目的を分離できている。

### ビジュアルデザイン

- 黒、金、白のコントラストでブランド感が明確。
- ヒーロー、カード、価格表、記事カードのトーンが一貫している。
- 画像アセットの使い方が適切で、プロダクトの用途に近い雰囲気を出せている。
- CTA は大きく、見た瞬間に押せる要素として認識しやすい。

### レスポンシブ

- 主要ページで水平スクロールは確認されなかった。
- モバイルではナビゲーションがハンバーガーに切り替わり、ヘッダーの破綻はない。
- 価格カードや法務テーブルは狭い幅でも読めるように収まっている。

### SEO

- prerender により 26 ルートが静的 HTML として生成されている。
- ブログ記事のテーマ選定は検索需要と相性がよい。
- 各主要ページに H1 があり、ページ単位の主題は明確。
- sitemap、robots、OG 画像など公開サイトに必要な基礎が揃っている。

## 優先して直すべき問題

### 1. モバイルトップの初期表示が重く見える

実ブラウザで `/` をモバイル表示したところ、ロード直後の撮影ではファーストビューの大部分が暗い背景のみになった。1.5 秒後には H1 と CTA が表示されるためレイアウト崩壊ではないが、初回訪問では「何も表示されていない」と誤認される可能性がある。

影響:
- 離脱率に直結する。
- SEO の Core Web Vitals 以上に、体感品質で損をする。
- CTA が見えるまでの時間が長く、配布サイトとしての目的達成が遅れる。

推奨:
- モバイルではヒーロー導入アニメーションを短縮する。
- H1 と主要 CTA は初期描画から表示済みに近い状態にする。
- 背景演出よりも「無料プランを試す」「使い方を見る」の即時視認性を優先する。

### 2. スクロール時の FPS が予算未達

`npm run audit:perf` で以下の結果になった。

- FCP: 612ms
- LCP: 612ms
- CLS: 0
- Long task total: 218ms
- Average FPS: 30
- Minimum FPS: 22.9

初期表示の数値は良いが、スクロール体験が重い。背景、グリッド、影、ぼかし、長いページ構成、演出の重なりが影響している可能性が高い。

推奨:
- モバイルでは背景固定、重い blur、過剰な shadow、スクロール連動演出を削減する。
- `prefers-reduced-motion` だけでなく、低幅 viewport 向けの軽量スタイルを用意する。
- トップページの長さとセクション数を見直し、重要情報を前半に寄せる。

### 3. 料金ページの購入判断はできるが、購入行動への確信がまだ弱い

料金ページは見た目が強く、3 プラン比較もわかりやすい。ただし「Free / Standard / Pro の違い」は読める一方で、ユーザーが自分の状況に合わせてどれを選ぶべきかの判断補助がもう一段ほしい。

推奨:
- 「週1本投稿なら Standard」「個別テンプレートが必要なら Pro」のような選び方を明文化する。
- Free から Standard へ移る具体的なタイミングを示す。
- 購入前によくある不安、解約、支払い、商用利用、サポート範囲へのリンクを価格表直下に置く。

### 4. トップページの H1 が長く、強いが理解に時間がかかる

H1 は内容を正確に伝えているが、かなり長い。デスクトップでは迫力がある一方、モバイルでは複数行に分かれ、認知負荷が高い。

推奨:
- H1 は「動画制作の前工程を、ひとつの運用線に」程度まで圧縮する。
- 対応サイト取得、台本整理、AI 補助、YMM4 前準備はサブコピーや機能列に分ける。
- SEO 用の詳細文は description と本文に任せる。

### 5. ブログは良いが、記事一覧の検索体験をさらに伸ばせる

ブログページは視覚的に強く、タグも用意されている。だが記事を探すユーザーに対して、検索ボックスとタグが少し同列で、目的別の入口が弱い。

推奨:
- 「初心者」「収益化」「素材」「YMM4」「権利」のようなカテゴリ導線を明確にする。
- Editor's Pick は良いので、次に読むべき記事の連鎖を記事詳細側にも強める。
- ブログトップのファーストビューで、製品導線への自然な接続を追加する。

## アクセシビリティ評価

良い点:
- モバイルで文字は概ね読める。
- CTA サイズはタップしやすい。
- ナビゲーションは狭い幅で破綻していない。
- コントラストは全体的に強めで、主要テキストは読みやすい。

懸念:
- 金色の小さいラベルや薄い説明文は、背景によってはコントラストが不足しやすい。
- アニメーション表示待ちがあるため、ユーザーによっては空白や遅延に見える。
- 価格ページや法務ページの長文は、モバイルでは読めるが疲れやすい。

推奨:
- 小さいラベルの明度差を少し上げる。
- モバイルでは行間を維持しつつ、カード内テキストの密度を少し下げる。
- `prefers-reduced-motion` 時の演出抑制を実機表示で再確認する。

## パフォーマンス評価

良い点:
- FCP / LCP / CLS は良い。
- ルート分割と prerender が効いている。
- 画像は WebP 中心で、公開サイトとして妥当。

懸念:
- CSS が 180.75kB とやや大きい。
- スクロール FPS が低く、体感で重くなる余地がある。
- トップページの `scrollHeight` が desktop で 15560px、mobile で 16414px と長く、最後まで読ませる設計としては負荷が高い。

推奨:
- ファーストビュー用 CSS と下層セクション CSS の整理。
- 装飾背景、グリッド、影、blur のモバイル軽量化。
- トップページのセクション圧縮、または重い演出の後半遅延。

## SEO 評価

良い点:
- 静的プリレンダー済みで検索エンジンに読ませやすい。
- ブログキーワードの設計が実用的。
- ページタイトル、H1、description の基本は揃っている。

懸念:
- トップページの訴求語がやや抽象寄りで、「ゆっくり動画 作り方」「YMM4 台本」「反応集 自動化」などの検索意図との接続をもっと前面に出せる。
- 製品の実画面、できること、対応範囲がファーストビューだけでは即断しにくい。

推奨:
- トップの早い位置に「対応サイト」「生成物」「YMM4 連携形式」「無料で試せる範囲」を表形式で置く。
- ブログ記事からダウンロード、使い方、FAQ への内部リンクを強める。
- FAQ の構造化データ導入を検討する。

## 次の改善優先度

1. モバイルトップのヒーロー表示を即時化する。
2. スクロール FPS 改善のため、モバイルの背景演出と shadow / blur を軽量化する。
3. トップページ H1 を短くし、機能詳細をサブコピーに分ける。
4. 料金ページに「どのプランを選ぶべきか」の判断ガイドを追加する。
5. ブログ詳細から製品導線への内部リンクを強める。
