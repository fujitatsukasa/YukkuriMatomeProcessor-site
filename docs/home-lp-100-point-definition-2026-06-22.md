# Home LP 100点定義 2026-06-22

## 目的

この文書は、ゆっくりまとめプロセッサーのHome/販売LPを「何を満たしたら100点と呼ぶか」を固定するための採点基準です。

今後は、この基準に照らしてだけ点数を出します。基準にない感覚点は出しません。

## まず結論

LPの100点は、見た目やコピーだけでは到達しません。

100点とは、次の3つが同時に成立している状態です。

1. 商品事実が確定している
2. LP上の表現とCTAがその事実だけを使っている
3. 実画面、配布、購入、法務、検証が矛盾していない

Codexだけでできるのは、主に2と3の実装・検証です。
1の商品事実の確定は、事業判断または実アプリ側の証跡が必要です。

## 採点できる前提

以下のどれかが未確定の間は、販売LPとして100点採点をしません。

- Freeでできることと上限
- Premiumで解除または追加される内容
- PC台数、PC変更、再認証ルール
- 更新範囲、メジャーアップデート時の扱い
- 対応URL、対応入力、対応YMM4バージョン
- 公開配布バージョン、署名状態、SHA256、配布ゲート
- 購入導線を本当に表示してよいか
- 実操作デモ、実画面、サンプル動画を証拠として使ってよいか
- 法務ページ、FAQ、料金ページ、アプリ内表示の一致

これらが未確定なら、できるのは「安全な未確定LP」までです。

## 100点ルーブリック

| 区分 | 配点 | 100点条件 | 主担当 |
| --- | ---: | --- | --- |
| 商品事実の確定 | 20 | Free/Premium、PC、更新、対応URL、配布、購入条件が公開可能な正本として確定している | 事業判断 + 実アプリ証跡 |
| 販売導線 | 15 | CTAがユーザー状態に合い、購入できる場合は購入へ、未確定なら確認へ正しく分岐する | Codex + 事業判断 |
| 証拠素材 | 15 | 実アプリ画面、実操作デモ、サンプルの位置づけが明確で、偽UIや未検証表現がない | Codex + 素材提供 |
| コピー/情報設計 | 15 | 購入前の疑問に先に答え、短く具体的な日本語で、誇大表現や内部語がない | Codex |
| 法務/信頼 | 10 | 価格、買い切り、返金、保証しないこと、権利確認責任が全ページで一致している | Codex + 事業判断 |
| UX/アクセシビリティ | 10 | PC/スマホで読める、横はみ出しなし、CTAが見つかる、フォーム/表/動画が破綻しない | Codex |
| SEO/schema/技術品質 | 10 | build、lint、構造化データ、OG、内部リンク、性能、画像alt、no broken CTAが確認済み | Codex |
| 外部レビュー耐性 | 5 | ChatGPTなど外部レビューに渡す素材、根拠、未確定扱いが揃っている | Codex |

合計: 100点

## 点数の上限ルール

未確定事項が残る場合は、満点ではなく上限を設けます。

| 状態 | 販売LPとしての上限 |
| --- | ---: |
| 購入条件、配布条件、Free/Premium条件が未確定 | 85点 |
| 購入条件は確定、配布条件または証拠素材が未確定 | 90点 |
| 商品事実は確定、実操作デモや証拠素材が弱い | 95点 |
| 商品事実、証拠、法務、CTA、検証が一致 | 100点 |

つまり、未確定事項を隠して強いコピーを書くことは加点ではありません。
それは100点ではなく、販売事故リスクです。

## 現在の扱い

2026-06-22時点の実装は、未確定条件を安全側に倒した状態です。

現在のLPは、次の方針です。

- 購入実行CTAは出さない
- 実行ファイルの直接取得CTAは出さない
- Premium価格、買い切り、月額なしは表示する
- Free/Premiumの具体上限は断定しない
- PC台数、再認証、更新範囲は断定しない
- 実画面は使うが、未検証の実操作証拠としては扱わない

この状態は「安全な公開前LP」としては正しいですが、「販売LP 100点」ではありません。

## 100点にするための未確定決定リスト

100点化に必要な決定事項は以下です。

| ID | 決めること | 決まったらCodexがやること |
| --- | --- | --- |
| D1 | Freeの上限 | 料金表、FAQ、Home、テストを確定値へ更新 |
| D2 | Premiumの上限/公正利用条件 | Premiumコピー、比較表、購入導線を更新 |
| D3 | PC台数 | FAQ、料金、法務補足、購入前注意を更新 |
| D4 | PC変更/再認証 | FAQ、購入前確認、サポート導線を更新 |
| D5 | 更新範囲 | 料金、FAQ、利用規約/特商法との整合確認 |
| D6 | 対応URL/入力範囲 | Home、FAQ、ブログ、schemaを更新 |
| D7 | 対応YMM4バージョン | FAQ、Download、Instructionsを更新 |
| D8 | 公開配布バージョン | Download、Update、構造化データを更新 |
| D9 | 署名/Windows警告の扱い | Downloadの安全確認、FAQ、信頼表示を更新 |
| D10 | 購入導線を出してよいか | purchaseReadyをtrue化し、購入CTAとテストを更新 |
| D11 | 直DL導線を出してよいか | downloadReadyをtrue化し、配布CTAとテストを更新 |
| D12 | 実操作デモ/動画サンプルの証拠利用可否 | Homeの証拠セクション、VideoObject、文言を更新 |

## Codexの次の動き

次は、採点ではなく未確定事項の回収に移ります。

1. この100点定義を正本にする
2. D1からD12の回答を埋める
3. 回答がある項目だけ `product-facts.ts` に反映する
4. CTAを出してよい条件が揃ったら購入/配布ゲートを解除する
5. Home、Purchase、Download、FAQ、Samples、Blogを横断更新する
6. copy、mobile、build、visual QAを再実行する
7. ChatGPT再レビュー用zipとプロンプトを作る

## 採点の言い方ルール

今後、Codexは次の形式でしか点数を言いません。

- 「この100点定義に照らすと、現在は何点」
- 「満点を阻んでいる未達項目は何か」
- 「その未達項目はCodexだけで直せるか、事業判断が必要か」
- 「次に何を決めれば何点上限が上がるか」

基準を示さない点数は出しません。
